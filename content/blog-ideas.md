# Blog Post Ideas

A running list of post concepts. Pick from here rather than forcing posts.
Aim for 2x/day to 1x/every 2-3 days — keep it active, not overwhelming.

---

## Ready to write

- **Rolling Home: Where It's Going** — honest look at the current gaps and the roadmap. Timers, counters, score tracker, turn order tracking, sequence card builder, duplicate templates. Frame around the promise the app makes and where it still needs to deliver.

## Worth writing when the time is right

- **Rolling Home: Sharing and Remote Play** — the sharing feature is a major update. Write this when it's getting close. The angle: Rolling Home was built for the same table, and what happens when it grows beyond that assumption.
- **What makes a good game template?** — design philosophy post about the decisions behind how Rolling Home structures turns, phases, and actions.
- **The QR code approach to game sharing** — if QR codes end up being the sharing implementation, that's a technical post worth writing.
- **Prism: Building a narrative engine** — when Prism has more shape, write about the design decisions behind perspective-driven storytelling.
- **Why I build in public** — meta post about the portfolio itself, the blog, the philosophy of showing work in progress.

## Miranor design posts (many more to come)

- **Controlled randomness** — the town store generation system. Each item has an `inStock` probability and a level requirement; when you enter town, every item rolls independently. The shop is never identical but never nonsensical. Angle: randomness isn't the opposite of design, it's a design tool. The guardrails are the design, the randomness is what makes the world feel alive inside them.

## Concepts / early ideas

- **Build the tool, then build the thing** — post about making tools that make your life easier. Anchor to the SVG-to-React converter (https://github.com/016spitfire/svg-to-react-converter) — a Node script that converts SVG files into ready-to-use React components. The story: SVG files don't drop cleanly into React (style attributes, kebab-case props, class vs className, etc.). Rather than do the conversion by hand every time, build the thing that does it for you. Pairs with a project page for the converter. Angle: the best tool you can build is the one that removes friction from your own workflow.

- **Necessity is the mother of invention** — Spencer's dad's line, how it shaped his design philosophy, how it shows up in everything he builds. Personal and grounded. Connects biography to practice.
- **Software is a living thing, not a shipped product** — the mindset behind active development, why "finished" is the wrong frame for software. Might be article-worthy instead.

- The remote tabletop question — can Rolling Home grow beyond the physical table? Probably too early to write this.
- Game endings are a design problem, not just a technical one — the philosophy behind what belongs in an app and what doesn't.
